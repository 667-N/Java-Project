# Етап 1
# Назва проєкту
SpaceVault

# Проблема
Шанувальникам ретро-естетики складно шукати автентичну графіку (blinkies, stamps, icons, cursors, pixels, icon masks, dividers, etc.) для кастомізації профілів у SpaceHey, Rentry та Carrd, оскільки старі архіви часто закинуті, а елементи розкидані по всьому інтернету. Відсутність єдиної впорядкованої бази даних уповільнює та ускладнює процес персоналізації сторінок.

# Рішення 
Застосунок функціонує як структурований каталог та архів ретро-графіки, де елементи розбиті за категоріями, тегами та тематикою. Користувачі можуть легко шукати потрібні блінки чи марки, зберігати їх у власні колекції («валіза» або «сейф») та в один клік копіювати готовий код для вставки у свій профіль.

# Цільовий користувач
Активні користувачі SpaceHey, веб-дизайнери-аматори, колекціонери цифрової ретро-естетики та всі, хто хоче швидко знайти якісний візуальний контент для оформлення своїх профілів.

# Етап 2
# 1. Вимірювані обмеження
Продуктивність (Performance): Час завантаження головної сторінки каталогу та відображення перших 20 графічних елементів (Contentful Paint) не повинен перевищувати 1.5 секунди при швидкості інтернету 3G.
Доступність (Availability): Система повинна забезпечувати рівень доступності (uptime) не менше 99.5% протягом місяця.
Безпека (Security): Усі запити до API повинні виконуватися виключно через захищений протокол HTTPS. Дані автентифікації користувачів або сесії «Обраного» мають бути захищені від XSS-атак.

# 2. Дизайн ключових ендпоінтів (API Contracts)

### Ендпоінт 1: Отримання списку графічних елементів з фільтрацією

Запит: GET /api/v1/graphics
Параметри запиту (Query Params):
`category` (string, необов'язковий) — наприклад, `blinkies`
`tag` (string, необов'язковий) — наприклад, `retro`
Контракт відповіді (Response):

```json
{
  "status": "success",
  "data": [
    {
      "id": "g_001",
      "title": "Cool Retro Star Blinky",
      "category": "blinkies",
      "tags": ["retro", "star", "blue"],
      "image_url": "https://spacevault.com/assets/blinkies/star.gif",
      "codes": {
        "html": "<img src=\"https://spacevault.com/assets/blinkies/star.gif\" alt=\"Star Blinky\">",
        "markdown": "![Star Blinky](https://spacevault.com/assets/blinkies/star.gif)"
      }
    }
  ]
}
```

### Ендпоінт 2: Додавання елемента до обраного
Запит: POST /api/v1/favorites
Контракт запиту (Request Body):
```json
{
  "graphic_id": "g_001"
}
```
Контракт відповіді (Response):
```json
{
  "status": "success",
  "message": "Element successfully added to favorites",
  "favorites_count": 5
}
```

# 3. Тест-кейси у форматі Given/When/Then
### Тест-кейс 1: Фільтрація каталогу за категорією
Given (Дано): Користувач перебуває на головній сторінці застосунку, де відображаються всі доступні типи графіки.
When (Коли): Користувач натискає на вкладку або фільтр "Stamps".
Then (Тоді): Система оновлює список і відображає лише цифрові марки (stamps), приховуючи блінки та іконки.

### Тест-кейс 2: Копіювання коду для Rentry
Given (Дано): Користувач відкрив картку конкретного блінка й бачить доступні варіанти коду.
When (Коли): Користувач натискає кнопку "Copy Markdown Code" (для Rentry).
Then (Тоді): Система копіює відповідний Markdown-рядок у буфер обміну та показує спливаюче сповіщення "Код скопійовано!".

### Тест-кейс 3: Додавання елемента до обраного
Given (Дано): Користувач переглядає каталог і знайшов марку, яка йому сподобалася.
When (Коли): Користувач натискає на іконку "сердечка" (Додати в обране) на картці елемента.
Then (Тоді): Іконка змінює колір на заповнений, а лічильник обраних елементів у меню збільшується на 1.
